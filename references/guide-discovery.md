# DISCOVERY · 问题情境解构

- 实施协议：`references/stage_protocols/problem-intelligence/SKILL.md`
- 关键交付物：`问题分析.md`
- 核验点：`approve`

## 主控职责

- 按“抽取题面、附件材料、用户补充”的证据位阶确认输入。
- 统筹子问题拆分、变量定义、约束识别、图形规划和工作计划。
- 闭合次序固定为 `validate -> gate_check -> complete -> checkpoint`。

## 实施角色

- `problem-reader`：查阅题面和附件，建立可信问题底稿。
- `problem-analyst`：形成子问题、变量、约束和候选建模方向。
- `enhancement-checker`：判定经典套路是否需升级为更贴合题意的竞赛方案。

## 审计重点

- 题面信息严禁被臆测替代。
- 子问题数量、假设预检、逐句审计与图形计划务必完整。
- `问题分析.md` 是后续阶段唯一的问题契约入口。
