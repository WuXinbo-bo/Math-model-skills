# 数学建模竞赛避坑大全

> 适用阶段：全流程（S1-S10）
> 来源：历年国赛评委反馈 + 获奖论文反面案例 + 社区经验总结
> 用途：Agent 在每个阶段检查是否踩坑，及时修正

---

## 一、致命级错误（P0，直接导致落选）

### 1. 跑题/偏题
**症状**：解的不是题目要求的，或者漏解子问题
**常见原因**：未逐字审题，忽略约束条件；把"优化"理解成"预测"；漏掉隐含子问题
**防范**：S1 阶段必做逐字审题 checklist

### 2. 结论与正文矛盾
**症状**：摘要说"精度99%"，正文只有85%
**常见原因**：摘要先写，正文后改了方法；代码改了但摘要没更新
**防范**：S10 终审必做 frozen_numbers 校验

### 3. 编造数据/参考文献
**后果**：直接取消评奖资格，且可能被通报
**铁律**：所有数据必须来自代码运行结果，所有参考文献必须真实存在

---

## 二、严重级错误（P1，显著扣分）

### 4. 摘要质量差
**正确做法**：摘要四要素 = 问题(1句) + 方法(具体算法名) + 结果(关键数值) + 验证(鲁棒性结论)
**字数控制**：300-500字

### 5. 图表无正文解读
**正确做法（D-A-C 三段论）**：
- D (Data): 描述图表展示了什么数据
- A (Analysis): 分析数据的模式、趋势、异常
- C (Conclusion): 得出该图表支持的结论

### 6. 无灵敏度/鲁棒性分析
必做：参数灵敏度（±10%/±20%/±50%）、基线对比、极端场景测试、脆弱结论标注

### 7. 格式严重不规范
字体仅用：宋体 + 黑体 + Times New Roman；行距统一：1.5倍或固定值22pt；页边距：上下左右均2.5cm

---

## 三、中等级错误（P2，扣3-5分）

### 8. 问题分析不够深入
应包含：题目信息梳理→子问题依赖关系→数据探索→方法初筛→难点预判

### 9. 模型假设不合理
好的假设：每条有必要性、有影响范围、数量合理（5-8条）、与实际不矛盾

### 10. 符号说明不完整
符号表应包含：所有正文出现的符号、含义和单位、跨子问题一致、按类别分组

---

## 四、常见陷阱（按阶段分类）

### S1 选题阶段
| 陷阱 | 应对 |
|------|------|
| 选题偏难 | 选有把握的，不选看着酷的 |
| 审题不细 | 逐字审题，列出所有约束 |
| 未分析数据 | 先做数据探索性分析 |

### S2-S4 建模阶段
| 陷阱 | 应对 |
|------|------|
| 过度建模 | 简单模型能解决就用简单的 |
| 模型不匹配 | 查 algorithm-selection-tree |
| 符号混乱 | 先建全局符号表 |

### S5-S6 求解阶段
| 陷阱 | 应对 |
|------|------|
| 代码不可复现 | 保存数据+环境+随机种子 |
| 结果不合理 | 先检查数据类型和量级 |
| 计算量过大 | 降采样/分块/近似 |

### S7-S8 验证阶段
| 陷阱 | 应对 |
|------|------|
| 跳过验证 | 必须做灵敏度分析 |
| 验证不充分 | 按模型类型做完整检查 |
| 忽略基线 | 基线对比是必选项 |

### S9-S10 论文阶段
| 陷阱 | 应对 |
|------|------|
| 摘要空洞 | 按模板写，四要素齐全 |
| 图表堆砌 | 每图必有D-A-C解读 |
| 结论过强 | 用限定性表述 |
| 字数不足 | 补充推导/对比/分析 |

---

## 五、紧急应对方案
### 时间不够用（最后6小时）
1. 摘要必须完整（方法+结果+结论）
2. 每个子问题必须有答案
3. 至少1张图表有D-A-C分析
4. 参考文献格式统一
5. 格式检查（字体/行距/页码）

### 结果不理想
诚实报告 + 讨论局限性 > 编造好看的结果

### 代码跑不通
检查数据格式→检查维度匹配→简化模型到能跑通→在论文中说明简化原因
