# 建模-代码一致性自检（全部题型必做）

**关键问题**：建模报告写了完整模型, 但代码实现时偷偷简化（用更简单的方法/忽略约束/降维）。

**关键问题**：题目给了具体数值约束（预算/时间/容量/数量等）, 但代码结果违反这些约束。

## 环节 1：建模-代码契约对照

⛔ 从 建模报告.md 提取以下"契约", 编码时务必逐条兑现：

⛔ 首要的是确认建模报告末尾的 5 项必备内容已经读过：

1. **结果约束清单** → 用于 validate_constraints(), 超出即代码有误

2. **预期行为描述** → 用于判定结果"形状"是否合理（稳态/瞬态/单调性）

3. **异常处理预案** → 遇到异常仅可按预案操作, 严禁自行发明修正方法

4. **方法唯一性声明** → 各个环节仅可用指定方法, 禁止替代

5. **验证检查点** → 代码跑完后逐项 pass/fail 核验

```bash

echo "=== 建模-代码一致性契约 ==="

echo "从 建模报告.md 提取关键承诺："

echo ""

# 1. 算法承诺

echo "--- 算法承诺（建模报告说用什么算法, 代码就必须用什么算法）---"

grep -i '算法\|方法\|求解\|使用.*法\|采用' 建模报告.md | head -20

# 2. 约束承诺

echo ""

echo "--- 约束承诺（建模报告列的约束, 代码必须全部实现）---"

grep -i '约束\|s\.t\.\|subject to\|≤\|≥\|不超过\|至少\|必须满足' 建模报告.md | head -20

# 3. 物理参数承诺

echo ""

echo "--- 物理参数（建模报告定义的参数值, 代码必须一致）---"

grep -oE '[A-Za-z_]+\s*=\s*[0-9.]+' 建模报告.md | head -20

# 4. 结果预期范围

echo ""

echo "--- 结果预期范围表（代码跑完后必须对照验证）---"

sed -n '/结果预期范围/,/^##/p' 建模报告.md | head -20

```

⛔ **一致性准则（硬性, 不可变通）：**

1. **算法不可降级**：建模报告说"遗传算法", 代码不可偷偷换成"贪心"; 说"SAT碰撞检测"不可换成"中心距判定"

2. **约束不可遗漏**：建模报告列了 N 个约束, 代码务必实现 N 个（不可"为简化"省掉几个）

3. **参数不可篡改**：建模报告定义 L=2.20m, 代码里不可写 L=1.65（用中间量代替）

4. **精度不可降低**：建模报告说"自适应步长 rtol=1e-8", 代码不可改成固定步长 dt=0.1

⛔ **若编码时发现建模报告的方案确实无法实现**（如算法太慢/库不支撑）, 务必：

- 在 计算结果.md 中清晰标明阐明"建模报告方案X无法实现, 缘由是Y, 替代方案是Z"

- 替代方案的精度不可比原方案差

- 不可静默替换——务必明示声明

## 环节 2：题目数值约束对照

题目一般情况下给具体数值约束（预算/时间/容量/数量等）, 跑完代码后要回到赛题校验：

```bash

echo "=== 题目约束对照 ==="

echo "--- 从赛题中提取数值约束 ---"

for src in 问题分析.md 用户数据/*_extracted.txt; do

    [ -f "$src" ] || continue

    echo "来源: $src"

    grep -oE '(不超过|最多|至少|不低于|大于|小于|≤|≥|<|>|=)[^。, ,.\n]{0,40}[0-9]+[^。, ,.\n]*' "$src" 2>/dev/null | head -20

    echo ""

done

```

若识别到约束违反：

- ❌ 禁止做法：把违反的值人工改小/改大让报告看起来合规

- ✅ 无误做法：回去调整优化模型（加约束 / 改目标函数 / 调求解方法）重新跑

## 环节 3：自动化约束校验（写进各个子问题代码末尾）

在各个子问题的代码末尾, **务必**加入以下约束校验代码块。这不是可选的"人工核验", 而是代码的一部分：

```python

# ===== ⛔ 约束验证（每个子问题必须有, 不可删除）=====

def validate_constraints(results: dict, constraints: dict) -> bool:

    """

    results: 计算结果字典, 如 {'gap': 0.081, 'velocity': -3.5}

    constraints: 约束字典, 如 {'gap': (0, 0.06, '悬浮间隙'), 'velocity': (-10, 10, '速度')}

                 格式: {key: (min_val, max_val, description)}

    返回: True=全部通过, False=有违反

    """

    violations = []

    for key, (lo, hi, desc) in constraints.items():

        val = results.get(key)

        if val is None:

            continue

        import numpy as np

        vals = np.atleast_1d(val)

        for v in vals:

            if v < lo or v > hi:

                violations.append(f"❌ {desc}: {v} 超出范围 [{lo}, {hi}]")

    if violations:

        print("\n" + "="*60)

        print("⛔⛔⛔ 约束验证失败 — 必须修正后才能继续")

        print("="*60)

        for v in violations:

            print(f"  {v}")

        print("\n修正要求：")

        print("  1. 分析超出原因（数据漂移/模型缺约束/边界效应）")

        print("  2. 在代码中加入物理约束（如接触约束、饱和限幅）")

        print("  3. 重新计算, 确保结果在约束范围内")

        print("  4. 不要删除此检查代码！")

        print("="*60 + "\n")

        return False

    else:

        print("✅ 约束验证通过")

        return True

# ⛔ 使用方法（根据题目实际约束填写）：

# constraints = {

#     'gap': (0, 0.06, '悬浮间隙(m)'),           # 题目明确给出最大间隙0.06m

#     'displacement': (-0.06, 0.06, '位移(m)'),   # 物理约束

# }

# passed = validate_constraints(results_dict, constraints)

# if not passed:

#     raise ValueError("约束验证失败, 必须修正代码后重新运行")

```

⛔ **强制准则**：

1. 各个子问题的代码文件末尾务必有 `validate_constraints()` 调用

2. 约束字典务必从 问题分析.md 和 建模报告.md 的"结果预期界限表"中提取

3. 若校验失败（返回 False）, 禁止留存 JSON、禁止继续下一个子问题

4. 务必调整代码加入物理约束后重跑, 直到校验通过

⛔ **AI 常见逃避行为（全部禁止）**：

- ❌ "数学上无误因而不需约束" → 错！物理约束 > 数学正确性

- ❌ "这是题目数据的特性" → 错！数据有问题就修正数据处理方式

- ❌ "纯数学ODE求解的结果" → 错！ODE 也务必加物理约束（接触、饱和、边界）

- ❌ 移除或注释掉 validate_constraints 代码 → 严重违规

- ❌ 把约束界限改大让结果"通过" → 篡改约束, 严重违规

## 环节 4：物理参数引用准则

代码中涉及碰撞检测、约束校验、目标函数计算时：

**务必采用 建模报告.md 中定义的完整物理参数**（如板凳全长220cm、矩形宽30cm、车身长4.5m）,

**禁止直接复用上游环节为其他目的计算的中间量**（如孔中心距165cm、把手坐标间距、质心偏移量）作为物理实体的几何代理。

**强制准则**：

1. 各个约束判定函数的文档字符串中务必注明其引用的物理参数来源及数值

2. 物理参数务必在代码顶部统一定义（从 建模报告.md 提取）, 不可散落在各处硬编码

3. 约束函数采用的参数值与 建模报告.md 不一致时, 务必抛出异常而非静默实施

4. 禁止"就近取值"——不可由于某个中间变量数值接近就拿来当物理参数用

**典型错误示例**：

- 板凳碰撞检测用"孔中心距165cm"代替"板凳全长220cm" → 错误（少算了两端各27.5cm）

- 车辆避障用"质心距离"代替"车身外轮廓最近点距离" → 错误（可能已经碰撞）

## 环节 5：未经校验的几何简化

若 建模报告.md 中标注了"待校验近似"的简化假设, 代码中务必：

1. 与此同时实现精确版本和简化版本
