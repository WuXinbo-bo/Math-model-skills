# 自检清单总索引（每问跑完后必读）

各个子问题代码跑完、结果 JSON 落盘后, 立即按下面过程读相应自检文件。

## 通用过程（全部题型）

1. 先 Read `consistency.md`（建模-代码契约 + 物理参数 + 结果区间）

2. 再 Read `sanity_check.md`（自动化数值复核 + 9 问背景复核）

3. 依据本子问题的题型, 选 Read 一个分类自检：

   - 优化类（调度/选址/路线/分配/规划/求最优值） → `optimization.md`

   - 预测类（时间序列/回归/分类） → `prediction.md`

   - 评价类（TOPSIS/AHP/熵权法/排名打分） → `evaluation.md`

   - 物理/几何类（碰撞/动力学/ODE/SAT 检测） → `physical.md`

   - 评价/统计/图论 → 在 `sanity_check.md` 末尾的 E/S/G 区段

4. 对照本资料每条 ✅/❌ 给出判定, 任何 ❌ 都务必改代码重跑

## ⛔ 触发硬性修复的判定准则（不可变通）

- ✅ = 结果完全在约束/合理界限内, 无需任何解释

- ⚠️ = 结果在边界附近但未超出, 需阐明为什么是合理的极端工况

- ❌ = 结果超出了题目清晰标明给定的约束/边界

**禁止的"假通过"行为**：

- ❌ 超出边界但标 ✅ 并写"已在报告中讨论" → 这是 ❌

- ❌ 超出边界但标 ✅ 并写"数学解超出物理界限" → 这是 ❌

- ❌ 超出边界但标 ⚠️ 并写"物理上可解释" → 超出就是 ❌

- ❌ 标了 ❌ 但不调整代码, 只在文字中"讨论" → 务必改代码重跑

## ⛔ 修复循环上限

同一个子问题最多修 3 轮。3 轮后仍不合理, 务必回退到建模阶段重新设计模型

（调整 建模报告.md）, 而不是在错误的模型上反复打补丁。
