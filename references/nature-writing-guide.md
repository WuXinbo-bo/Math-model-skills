# Nature-Style 数学建模论文写作指南

> 集成自 `nature-writing` skill + 2026 CUMCM 评分标准 + 反 AIGC 检测实践。
> 最后更新：2026-06-16

---

## 核心原则
1. **先有论据，再有句子。** 不编造结果、机制、参考文献。
2. **每段一个信息。** 段首句声明段落目的，后续句展开。
3. **主张要有边界。** 不用"显著提高"而不给基线，不用"首次"而不限范围。
4. **观察归结果，含义归讨论。** 不把相关性当因果。
5. **用数字说话，不用形容词包装。** "精度从 0.72 提升到 0.89"优于"精度显著提升"。
6. **诚实优于完美。** 承认局限性比夸大结论更得分。

## 一、2026 反 AIGC 检测写作规范（P0 级）

### 1.1 AI 写作痕迹黑名单（34 类必须消除）

**套话类（P0，发现即扣分）：**

| AI 痕迹 | 问题 | 正确做法 |
|---------|------|---------|
| `本文针对...问题，建立了...模型` | 千篇一律开头 | 直接切入技术难点："XX 问题的核心矛盾在于 YY" |
| `首先...其次...最后...` | 流水账结构 | 改为"挑战→方法→证据"逻辑链 |
| `具有重要的理论意义和工程价值` | 空洞拔高 | 具体说明"可使调度误差从 8.2% 降至 3.1%" |
| `综上所述/总而言之/概括而言` | 万能总结词 | 直接写结论，不加"综上" |

**过度修饰类（P0）：**
| AI 痕迹 | 问题 | 正确做法 |
|---------|------|---------|
| `显著提高` | 懒 | 给基线比例："从 0.72 提升到 0.89" |
| `大幅度提升` | 无量 | 具体数值 |
| `充分考虑` | 不可验证 | 列出考虑了哪些因素 |
| `进行了深入研究` | 自夸 | 说明研究范围 |
| `完美解决` | 绝对化 | 限定条件 |

**模糊量化类（P0）：**
| AI 痕迹 | 问题 | 正确做法 |
|---------|------|---------|
| `效果良好` | 未量化 | 给具体数值 |
| `表现优异` | 未量化 | 给具体数值 |
| `精度较高` | 未量化 | 给误差指标 |
| `误差较小` | 未量化 | 给出"RMSE = 0.23" |
| `效果显著` | 未量化 | 给出 p 值或效应量 |

**万能连接词类（P0）：**
| AI 痕迹 | 问题 | 正确做法 |
|---------|------|---------|
| `值得注意的是` | 废话 | 删掉或直接陈述 |
| `不难发现` | 废话 | 删掉或直接陈述 |
| `众所周知` | 假设已知 | 给出依据或引用 |
| `在此基础上` | 指代模糊 | 明确"在 XX 方法基础上" |
| `进一步分析表明` | 废话 | 直接陈述发现 |

**因果断言类（P0）：**
| AI 痕迹 | 问题 | 正确做法 |
|---------|------|---------|
| `实验证明` | 过度声称 | 改为"数据表明 X 与 Y 相关" |
| `结果充分证明` | 绝对化 | 限定"在测试条件下" |
| `毋庸置疑` | 过强 | 改为"结果高度提示..." |
| `无可辩驳的证据` | 超出范围 | 改为"数据支持 XX 结论" |

### 1.2 结构性 AI 痕迹检查
- [ ] 句长是否有变化（禁止连续 3 句以上相同长度）
- [ ] 段落长度是否有变化（禁止连续 3 段以上相同长度）
- [ ] 段落结构是否有变化（论证型/叙述型/对比型混合）
- [ ] 无平行结构三连
- [ ] 每段表达方式是否独特

### 1.3 反 AI 写作策略
#### 策略 1：句长变化
```
不好：连续 3 句相似长度
  "本文建立了优化模型。该模型考虑了多个约束条件。求解采用遗传算法。结果表明效果良好。"

好：长短句交替
  "核心问题是优化。我们设计了一个含七项约束的模型，涵盖成本、距离和容量限制——找到合适的平衡是主要难点。遗传算法在两分钟内完成求解。"
```

#### 策略 2：段落结构变化
- 不要总用"首先、其次、第三"结构
- 交替使用：问题→方法→证据；例子→规则→例外；一般→具体→总结

#### 策略 3：词汇多样性
- 避免重复使用同一种句式结构
- 使用不同的过渡方式（因果、对比、举例、顺序）
- 同一万能短语在全文中不超过1次

#### 策略 4：知识深度
- 对问题的理解越深入，AI 模式越少出现
- 尽量用自己的话描述方法而非照抄定义
- 插入具体的问题领域细节：精确数值、边界情况、实际误差分布

## 二、各章节写法规范

### 2.1 摘要（四要素）
**要素1 背景**（1句）：解决什么问题？为什么重要？
**要素2 方法链**（2-3句）：每个子问题用了什么方法？为什么？
**要素3 关键数值结果**（3-5句）：每个子问题得到了什么具体数值？
**要素4 验证+总结**（1句）：验证方法/鲁棒性 + 总体结论

**格式**：每子问题用句式："针对问题X，考虑……，建立……模型，采用……算法，得到……结果。"

**不可漏的**：
- 每子问题 ≥1 个量化结果（含对比数据）
- 数值可追溯：每个数字都能在正文中找到
- 无模糊词：大概/可能/也许/显著/明显

### 2.2 问题重述（倒推法）
- 不是：照抄题目原文
- 结构：什么背景 → 要做什么 → 分几个子问题 → 有什么约束
- 技巧：简单问题用简化语言重述，难点问题点出核心

### 2.3 问题分析（五层法）
1. 题目信息梳理（已知/未知/约束）
2. 子问题间依赖关系
3. 数据探索（数据长什么样？）
4. 方法选择理由（考虑哪些模型？优劣？）
5. 难点预判（哪里可能卡住？）

### 2.4 模型假设（标准结构）
每条假设必须包含：
```
(H1) [假设名]：[假设内容]。必要性：[必要/简化]因为[理由]。范围：[全局/Qx]。替代：[可放松方式或灵敏度分析]。
```
格式：三列表格（描述+论证+条件）或结构化列表

### 2.5 模型建立（五步法）
1. 问题形式化：用数学语言重述
2. 建模思路：为什么选这类模型？替代方案？
3. 变量与参数定义：符号表
4. 目标函数/控制方程推导：从基本原理出发
5. 约束条件/边界条件：逐个写明含义

### 2.6 模型求解（三步）
1. 求解过程：如何求解？参数选择依据？
2. 核心结果：数字+含义+解释
3. 交叉验证/鲁棒性：指标+含义

### 2.7 模型评价（五维）
1. **优点**：2+条（数据支撑，不自夸）
2. **缺点**：2+条（诚实，不敷衍）
3. **推广方向**：条件允许如何扩展
4. **与已有工作对比**：新在哪里
5. **实际可行性**：真要用需要多少计算资源

### 2.8 结论（三段式）
1. 总结：逐题回答，复述关键数值
2. 局限：客观陈述（与评价段不同）
3. 展望：未来研究方向（与评价段的推广不同）

### 2.9 参考文献（格式）
- CUMCM：GB/T 7714-2015
- 期刊：[编号] 作者, 论文名, 期刊名, 卷(期): 年, 页码.
- 书籍：[编号] 作者, 书名, 出版地: 出版社, 出版年.

## 三、段落流畅性检查
- [ ] 段落之间过渡自然（没有突兀的话题切换）
- [ ] 每段服务于一个明确目的
- [ ] 跨段落结论相互支撑（证据累积）
- [ ] 文字与图表交替出现（不是一整面文字墙）
- [ ] 阅读流畅：无需回读就能理解论证

---

## 四、图表标题标准
**错误标题（描述性）**：
```
表3：不同模型的精度对比
图4：不同温度下预测误差的变化
```

**正确标题（结论性）**：
```
表3：模型A准确率达94.7%，优于所有基线至少12%
图4：25°C时预测误差最小(RMSE=0.23)，超过±15°C后线性增大
```

## 五、提交前自查清单
1. **摘要**：四要素齐全？每子问题≥1个量化结果？无模糊词？
2. **数值**：摘要中每个数值与正文一致？可追溯至 frozen_numbers.json？
3. **图表**：每张图有 D-A-C？标题是结论性的？
4. **假设**：3-5条？每条有论证？区分了必要和简化？
5. **模型评价**：2+优点+2+缺点？有数据支撑？
6. **格式**：章节层级正确？参考文献格式统一？无占位符？
7. **规范**：字体统一？三线表正确？公式编号连续？
8. **AI痕迹**：无34类套话？句长有变化？段落结构有变化？
