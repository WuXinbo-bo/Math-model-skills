# 数学建模竞赛论文写作规范 v4.0

> **适用**：所有论文生成阶段。Agent 在 `paper_write` 和 `content_assembly` 阶段**必须**遵循本规范。
> **来源**：综合 KyrieZhang329/MathModeling-skills 论文写作规则 + G5.1/G5.2 质量门禁

---

## 一、基本规则

### 从产物写，不从记忆写
论文每个段落必须有对应的上游产物支撑：

| 论文部分 | 对应产物 |
|---------|---------|
| 问题重述 | problem_parse.json |
| 问题分析 | problem_classification.json + question_dependency.md |
| 模型假设 | model_assumptions.md |
| 符号说明 | global_symbol_table.md |
| 数据预处理 | data_report.md |
| 模型构建 | final_method_explanation.md |
| 模型求解 | 代码 + run_summary.json |
| 结果分析 | final_result_analysis.md |
| 鲁棒性分析 | robustness_report.md |
| 图表 | figure_table_plan.md + 图源文件 |

### 禁止编造
永远不要编造：数据、数值结果、参考文献、图表、参数值、实验结果、模型性能声明。

---

## 二、文本质量标准

### 2.1 字数要求（G5.1 强制）
| 章节 | CUMCM 最低字数 | 说明 |
|------|--------------|------|
| 摘要 | 300 | 不超过1页 |
| 问题重述 | 400 | 背景+已知+待解决 |
| 问题分析 | 500 | 总体+每子问题方法选择理由 |
| 模型假设 | 300 | 3-5条，每条有论证 |
| 符号说明 | 200 | 三列表格 |
| 模型求解（每子问题） | 1500 | 公式+推导+解释 |
| 统一框架 | 500 | 跨子问题联系 |
| 灵敏度分析 | 400 | 扰动+鲁棒性 |
| 模型评价 | 400 | 2+优点+2+缺点 |
| 结论 | 300 | 逐题回答 |
| **正文总计** | **12000** | 不含摘要和附录 |

### 2.2 每个数值 ≥3 维讨论（G5.2 强制）
每个关键数值结果必须包含以下至少3个维度的讨论：
1. **灵敏度**：参数变化时该值如何变化？
2. **物理含义**：该值在语境中实际意味着什么？
3. **基线对比**：与最简单方法比如何？
4. **交叉验证**：与独立方法结果一致？
5. **不确定性**：误差范围是多少？

### 2.3 引用规范
- 可接受：标准方法教材、期刊论文、会议论文、权威技术报告
- 禁止：维基百科、博客文章、课程幻灯片、ChatGPT 生成的参考文献
- 每个子问题至少 2 篇支撑参考文献
