# 冠军模式模拟审稿方法

本知识库仅在 `quality_mode=championship` 时加载，用于在 `MANUSCRIPT` 完成后对论文实施多轮独立质量审稿。审稿务必以磁盘证据为准，不可依据印象评分。

## 1. 审稿闭环

每轮固定实施：

1. 独立审稿者查阅论文和上游证据；
2. 交付带定位信息的 P0/P1/P2 问题清单；
3. 主控角色形成可实施修订计划；
4. 写作角色产出完整修订稿，不只提交补丁片段；
5. 独立复核者逐项校验修复结果；
6. 登记剩余问题和轮次得分。

审稿者与调整者务必职责分离。复核严禁仅引用调整日志，务必重新核验修订后的全文。

## 2. 三轮必审视角

### 第一轮：数学与模型严谨性

- 假设是否有依据、适用边界和失效条件；
- 变量、符号、单位、目标与约束是否完整一致；
- 推导是否跳步，模型选取是否有对照依据；
- 算法是否真实相应数学模型；
- 灵敏度、稳健性和误差分析是否支撑结论。

### 第二轮：证据与可复现性

- 论文数值是否能追溯到程序和结果文件；
- 摘要、正文、图形、表格与结论中的数字是否完全一致；
- 数据来源、预处理、参数和随机性是否透明；
- 是否出现数据泄漏、失效结果或无法复现的声明；
- 图形与表格是否真正支撑相邻论述。

### 第三轮：竞赛评委与表达质量

- 快速阅读时能否识别问题、方法、结果、创新和结论；
- 标题、摘要与结论是否形成清晰信息闭环；
- 是否出现空泛套话、过度宣称、重复段落或模板化表达；
- 章节结构、图形与表格叙事、公式编号和引用格式是否专业；
- 是否满足匿名、版式和竞赛提交标准。

需第四、第五轮时，分别聚焦领域常识反证和终版出版就绪核验。

## 3. 问题分级

- `P0`：会导致结论错误、诚信风险、关键数值矛盾、匿名泄露、证据链断裂或无法提交的问题。务必清零。
- `P1`：明显影响模型可信度、复现性、评委理解或论文专业度的问题。终版最多保留 2 项，且务必解释风险。
- `P2`：不改变正确性但影响可读性、精炼度或视觉一致性的问题。尽可能修复。

## 4. 评分结构

| 维度 | 权重 |
| --- | ---: |
| 数学与模型正确性 | 25 |
| 证据链与可复现性 | 20 |
| 跨章节逻辑一致性 | 15 |
| 结果可信度与校验 | 15 |
| 竞赛表达与结构 | 15 |
| 图形与表格、格式与合规 | 10 |

冠军通过条件：综合分 `>=85`、`P0=0`、`P1<=2`，且完成不少于三轮有效审稿。

## 5. 强制攻击策略

- 假设攻击：逐条询问假设在什么条件下失效。
- 推导攻击：核验公式之间是否缺少必要推理。
- 数据攻击：核验来源、处理偏差、泄漏和异常值影响。
- 结果攻击：核验量级、边界、基线改进和稳定性。
- 反例攻击：尝试构造使结论不成立的情景。
- 负结果诚实性：核验是否回避不显著或不理想结果。
- 评委速读攻击：只读摘要、图形、表格与结论，判定关键贡献是否仍成立。

## 6. 回退准则

若 P0 涉及模型假设、公式推导、算法实现或关键求解结果，严禁仅调整论文，应回退：

- 问题理解错误 → `DISCOVERY`
- 模型机制错误 → `FORMULATION`
- 程序或结果错误 → `COMPUTATION`
- 图形与表格证据错误 → `EVIDENCE` 或 `SCHEMATICS`

回退后务必再次实施受影响的下游阶段，再重新启动冠军审稿。

## 7. 每轮文件合同

每轮目录 `审查/冠军模式/第NN/` 务必涵盖：

- `审稿报告.md`：带位置、等级、证据和推荐的审稿报告；
- `修订计划.md`：逐项修订计划；
- `修订论文.tex`：完整修订论文；
- `修订复核.md`：逐项修复复核与剩余风险。

终版产出：

- `论文/冠军修订稿.tex`
- `审查/冠军模式/最终报告.md`
- `状态/冠军审稿.json`

终版稿会备份原 `论文/论文正文.tex` 后回写为新的 `论文/论文正文.tex`，供 `ASSURANCE` 编译验收。
