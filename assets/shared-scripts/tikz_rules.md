# TikZ 架构图产出准则

Meta-model-agent 产出 TikZ 架构图时参考此文件。模板是起点不是终点，鼓励依据具体研究内容自由发挥布局和视觉设计。但下面标注 ⛔ 的低级错误务必避免。

## ⛔ TikZ 适用界限（先判定再画）

TikZ 擅长：过程图、技术路线图、变量关系图、算法过程图（带公式）、因果路线图、网络拓扑图、指标体系图。

**⛔ 以下场景避免用 TikZ，改用 AI Image 或 DrawIO：**

- 复杂 3D 场景（圆柱体透视、球面反射、多层空间结构）→ 用 AI Image

- 写实物理/工程场景（无人机、传感器、交通场景、建筑物）→ 用 AI Image

- 简单的分层/分组/树形结构（无公式、无跨层连线）→ 用 DrawIO（更快更稳）

**TikZ 的优势场景（用 TikZ）：**

- 需 LaTeX 公式渲染的图（如 $\Delta f < \epsilon$）

- 需精确控制连线路线的图（跨层箭头+标注系数）

- 变量关系/因果路线图（三角/菱形布局+路线系数）

- 简单 2D 平面几何示意图（三角形、圆、坐标系、角度标注、虚线辅助线）

- 算法过程图（带判定分支和公式）

- 网络拓扑图（≤15 节点）

**配色准则：自由搭配，但要精致（竞赛论文除外，见下方竞赛配色准则）**

⛔ 避免每篇论文都用 `blue!80 + teal!70`。demo 文件里的配色只是示例，真实产出时务必依据论文主题自由搭配。

**全部配色方案都遵循模板 4 的浅色原则**：主节点 `fill=主色!6`（极浅）+ `draw=主色!80`（中等边框）+ 子节点 `fill=white` + `draw=辅色!70`。避免用深色填充。

**6 套配色方案**（按论文主题选，同一用户不同论文换着用）：

| 方案 | 主色（边框+箭头） | 主色填充 | 辅色（子节点边框） | 适合主题 |

|------|-----------------|---------|------------------|---------|

| A 学术蓝 | `blue!80` | `blue!6` | `teal!70` | 竞赛、通用、工程 |

| B 森林绿 | `green!70!black` | `green!5` | `teal!60` | 环境、生态、农业 |

| C 暖灰棕 | `brown!70` | `brown!5` | `orange!50` | 经管、社科、教育 |

| D 靛蓝紫 | `indigo!70` | `indigo!5` | `violet!50` | 医学、心理、生物 |

| E 青灰色 | `cyan!70!black` | `cyan!5` | `gray!50` | CS、AI、数据科学 |

| F 石板灰 | `gray!70` | `gray!4` | `gray!50` | 数学、物理、理论 |

**采用方式**：把模板 4 的样式定义里的颜色替换即可，结构不变：

```latex

% 方案 B 森林绿 示例

main/.style={..., draw=green!70!black, fill=green!5, ...},

sub/.style={..., draw=teal!60, fill=white, ...},

bigarrow/.style={-stealth, line width=1.4pt, color=green!70!black},

```

**⛔ 禁止的配色**：

- `red!80`、`orange!80`、`violet!80` 等高饱和暖色做主色 — 太刺眼

- 主节点 `fill=主色!30` 以上 — 太深，文字看不清

- 各个节点不同颜色 — 彩虹效果

**选色推荐**（不强制，可自创）：

| 论文主题 | 推荐方案 |

|---------|---------|

| 经管/金融 | 方案 A 或 C |

| 医学/健康 | 方案 D |

| 环境/生态 | 方案 B |

| 工程/CS | 方案 A 或 E |

| 社科/教育 | 方案 C |

| 数学建模竞赛 | 依据赛题主题从 A-F 中选 |

| 数学/物理 | 方案 F |

**⛔ 竞赛论文配色准则**：竞赛论文从 6 套配色方案（A-F）中依据赛题主题自由选取。举例而言生态类赛题用方案 B 森林绿，经济类用方案 C 暖灰棕，医疗类用方案 D 靛蓝紫。同一篇论文内全部 TikZ 图务必用同一套配色。避免各个问题换一个颜色，不同问题通过虚线框分组区分。

**务必保持的精致元素**（不管什么配色方案都要有）：

- `drop shadow={opacity=0.15}` 微阴影

- 矩形节点不得设置 `rounded corners`，统一使用直角；其他几何形状保持不变

- `fill=主色!5~!6` 极浅填充（不是纯白也不是深色，参照 6 套配色方案的填充列）

- `line width=0.7pt` 主节点边框粗细

- 子节点 `fill=white`，边框用辅色

- 虚线框统一 `draw=gray!40, dashed, fill=gray!2`（全部配色方案通用，避免变）

- 粗箭头颜色跟主色保持一致

**鼓励的创意方向**：

- 非对称布局（左重右轻、上密下疏）

- 圆弧箭头、曲线连接（`bend left/right`）

- 渐变色填充（`left color=...!10, right color=...!20`）

- 图标化节点（用简单符号增加识别度）

- 分组嵌套（大框套小框表示模块层级）

- 时间轴布局（横向时间线 + 上下事件节点）

## ⛔ 零容忍的低级错误（务必避免）

### 编译错误类

- ⛔ 禁止 `on background layer` — 缺少 `backgrounds` 库会导致黑底

- ⛔ 禁止 `fit=()` — 缺少 `fit` 库会编译失败

- ⛔ 禁止未闭合的数学模式 `$` — 各个 `$` 都要有配对

- ⛔ 禁止 `\sout`/`\cancel`/`\xcancel` 在 TikZ 节点内 — 渲染不稳定

- ⛔ 节点内用 `\\` 换行时务必加 `align=center` 或 `align=left` — 反之编译报错

- ⛔ 禁止 `\textbf{}` 在数学模式 `$...$` 内 — 用 `\mathbf{}` 或 `\boldsymbol{}`

- ⛔ 用了 `diamond` 节点务必保证加载了 `shapes.geometric`

- ⛔ 用了 `decorate` 务必保证加载了 `decorations.pathreplacing`

### 视觉灾难类

- ⛔ 禁止灰色大背景 `\fill` — 白底最安全

- ⛔ 禁止浅色文字 — 全部文字（节点内、标注、阶段标签、右侧注释）务必清晰可读。具体禁止：

  - `color=gray!30~70`、`color=主色!20~40`（如 `teal!30`、`blue!25`）— 打印后看不见

  - 右侧注释/阶段标签务必用 `color=black!70` 或更深，避免用 `color=teal!40` 之类的浅色

  - 若标注文字在白色背景上，颜色深度不少于 `!60`（如 `gray!60` 勉强可，`gray!40` 更安全）

- ⛔ 禁止白色文字 `text=white` 出现在浅色背景节点上 — 只有深色填充节点（如圆形编号、胶囊标签）才能用白色文字

- ⛔ 反馈/回路箭头务必走空白区域 — 若底部有标注文字，反馈箭头走上方（`.north` 出发）；若上方有标题，走下方但要在标注文字下方留足间距

- ⛔ 禁止随机彩虹配色 — 单张图内的配色要有逻辑（按阶段分色或按功能分色），避免各个节点随机一个颜色

- ⛔ 禁止纯蓝/纯红/纯绿高饱和填充 — 用低饱和色（`!10~!15` 填充 + `!50~!60` 边框）

### 重叠类

- ⛔ 节点间隙 >= 0.3cm — 任何两个节点边框不可紧贴或重叠

- ⛔ 中文文字宽度约等于 2 倍英文 — 节点 `minimum width` 和 `text width` 务必按中文字数计算

- ⛔ 侧边标注间距 >= 1.5cm — 多行标注不可重叠

- ⛔ 标注+公式混排用 `text width` — 防止溢出，准许自动化换行

- ⛔ 虚线框标签不可和标题重叠 — 垂直间距 >= 0.8cm

- ⛔ 密集连接线不超过 8 条 — 超过时用汇聚节点或省略号

- ⛔ 禁止不透明填充方块遮挡其他元素 — 如果需要标记点/区域，用 `fill opacity=0.3` 或 `draw` 边框而不是实心填充。任何 `\fill` 或 `fill=颜色` 的节点不能覆盖在文字、连线、其他节点上面。如果需要背景色块，用 `on background layer`（需要 `\usetikzlibrary{backgrounds}`）

- ⛔ 标注文字不可挤在一起 — 同一区域的多个标注（如角度 α、法向量 n̂、坐标点）务必错开放置，间距 >= 0.4cm。若空间不够，用引线（`--` + 小偏移）把标注拉到空白区域

### 箭头类

- ⛔ 箭头/连线绝对不可穿过任何节点或文字 — 这是最常见的丑图缘由。务必用以下方式绕行：

  - 垂直对齐的节点之间：直接垂直连线，避免从远处斜着拉过来

  - 需跨行连接时：先垂直走到行间空白区域，再水平拐弯（用 `|-` 或 `-|`）

  - 需绕过中间节点时：从节点外侧走（`++(偏移)` 先走出去再拐弯）

  - 反馈/跳跃连接：走图的最外侧（左侧或右侧边缘）

- ⛔ 虚线箭头不穿过文字 — 用 `bend left/right` 绕开

- ⛔ 若两个节点之间有一行工具/方法小节点挡着，连线务必从相应列垂直下来，避免从一个点分叉斜穿

### 尺寸类

- ⛔ 图不能超出页面宽度 — 横向节点超过 5 个时用 `scale=0.8` 或分两行，用 `\adjustbox{max width=\textwidth}` 兜底

- ⛔ 图不可超出页面高度 — 纵向超过 6 个阶段时用 `scale=0.75` 或压缩间距

## 推荐（非强制，提升美观度）

### 文字与标签

- 节点内文字用黑色或深灰色

- 左侧标签推荐 `color=black`

- 解释性文字写在正文里，避免放图里

- 节点文字用 `\footnotesize` 或 `\small`

- 中文标点在节点内尽可能避免，用空格或换行代替

### 尺寸控制

- 用 `scale=0.85` 控制大小，避免与此同时用 scale 和 adjustbox/resizebox

- 阶段数量 4-6 个，每阶段子节点不超过 5 个

### 布局技巧

- 虚线框坐标按公式计算（见 `figure_style_guide.md` 的虚线框坐标计算准则）

- 子节点超过 4 个推荐分两行，间距不少于 1.2cm

- 侧边标注超过 6 个字符时，用 `text width` + `align=left` 自动化换行

- 虚线框标签放在框的左上角外侧，字号用 `\scriptsize`

- 阶段间推荐用粗箭头，节点间用细箭头

- 箭头尽可能走水平或垂直，拐弯用 `|-` 或 `-|`

- 若需表示移除效果，用红色 `\draw` 线条覆盖在节点上方

### preamble 依赖

- 基础：`\usepackage{tikz}` + `\usetikzlibrary{arrows.meta, positioning, shapes.geometric, calc}`

- 大括号装饰需要 `\usetikzlibrary{decorations.pathreplacing}`

- 阴影效果需要 `\usetikzlibrary{shadows}`

### 自检清单（画完后过一遍）

1. 全部数学模式 `$` 都成对闭合了吗？

2. 用了 `\\` 的节点都有 `align=` 属性吗？

3. 用了 `diamond`/`decorate` 的图都加了相应的 `\usetikzlibrary` 吗？

4. 图的总宽度超过 14cm 了吗？超过就加 `scale` 或 `adjustbox`

5. 有没有箭头穿过了不有关的节点？

6. 中文标注的节点 `minimum width` 够宽吗？

### ⛔ 中文节点宽度计算公式（防遮挡关键）

中文字符宽度约为英文的 2 倍。节点宽度务必按以下公式计算：

```

minimum width = 中文字数 × 0.7cm + 英文字符数 × 0.35cm + 1.0cm（内边距）

```

示例：

- "数据预处理"（5 个中文）→ minimum width = 5×0.7 + 1.0 = 4.5cm

- "LSTM 模型"（4 中文 + 4 英文）→ minimum width = 4×0.7 + 4×0.35 + 1.0 = 5.2cm

- "空间杜宾模型 (SDM)"（7 中文 + 5 英文）→ minimum width = 7×0.7 + 5×0.35 + 1.0 = 7.65cm

**⛔ 若节点文字超过 8 个中文字符，务必用 `text width` + `align=center` 准许自动化换行，而不是加大 `minimum width`。**

### ⛔ 编译后自检（务必实施）

TikZ 图写完后，务必编译一次并运行自检脚本：

```bash

# 1. 编译检查 overfull

xelatex -interaction=nonstopmode 论文正文.tex 2>&1 | grep -i "overfull\|underfull\|badbox"

# 2. 运行 tikz_check.sh 自动检测（节点重叠+中文宽度+连线穿过+配色）

bash 工具/tikz_check.sh 图表/结构示意图.tex 2>/dev/null || bash skills/shared-scripts/tikz_check.sh 图表/结构示意图.tex

```

**⛔ tikz_check.sh 报告的全部 CRITICAL 务必修复后重新编译，不准许跳过。** 常见修复：

- 节点重叠 → 调整坐标间距（同行 ≥ 1.0cm，上下 ≥ 0.8cm）

- 中文宽度不足 → 加大 `minimum width` 或用 `text width` + `align=center` 换行

- 连线穿过节点 → 改用 `|-` 或 `-|` 直角拐弯，或加 waypoint 绕行

- Overfull hbox → 缩短文字、加大节点、或用 `\footnotesize`

若出现 `Overfull \hbox` 警告，阐明有文字溢出节点边界，需：

1. 加大相应节点的 `minimum width` 或 `text width`

2. 缩短节点文字（用缩写）

3. 用 `\footnotesize` 或 `\scriptsize` 缩小字号

**避免忽略 overfull 警告——它就是遮挡的信号。**

### ⛔ 全部 TikZ 图的防遮挡准则 + 精致度要求（模板 4/5/6/7/8/9/10 通用）

**精致度基线（全部 TikZ 图务必达到）**：

- 全部矩形节点务必删除 `rounded corners`；菱形、圆形、六边形、圆柱等语义形状保持不变

- 主节点务必有 `drop shadow={opacity=0.15, shadow xshift=0.5pt, shadow yshift=-0.5pt}`

- 主节点填充用极浅色（`fill=主色!6`），不是纯白也不是深色

- 子节点/工具节点用 `fill=white` + 辅色边框

- 虚线框用 `draw=gray!40, dashed, fill=gray!2`

- 箭头用 `-stealth`，粗箭头 `line width=1.4pt`，细箭头 `line width=0.5pt`

- **模板 4 的间距标准是全部模板的参考基线**：虚线框 13cm 宽、子节点 x 间距 2.8cm、主节点和子节点 y 间距 1.0cm

技术路线图是遮挡重灾区，由于节点多、层级深。以下准则务必遵守：

**1. 同一行的节点间距 ≥ 1.0cm（边框到边框）**

- 3 个节点一行时，x 坐标间距不少于 4cm（如 3, 7, 11）

- 工具/方法小节点一行时，x 坐标间距不少于 2.2cm

- 若一行放不下，分两行，避免硬挤

**2. 上下层之间间距 ≥ 0.8cm**

- 方法节点行 → 工具节点行：y 间距不少于 1.0cm

- 工具节点行 → 下一个方法节点行：y 间距不少于 1.2cm

- 阶段间粗箭头长度不少于 0.6cm

**3. 长文字节点务必加宽或换行**

- "稳健性与内生性检验"（9 个中文）→ 务必用 `text width=4cm` + `align=center`

- 或缩写为"稳健性检验"（5 个中文）

- 工具节点文字不超过 5 个中文字符

**4. 禁止装饰性元素遮挡内容**

- 避免加灰色大背景 `\fill`（白底最安全）

- 底部方法论标注栏若和最后一个阶段太近（< 0.5cm），就避免加

- 右侧反馈环虚线若和交付标签重叠，就避免加

- 卡片背景色务必极浅（RGB 各个通道 ≥ 240），反之节点文字看不清

**5. 模板内容务必契合论文主题**

- 模板中的示例文字（"研究问题提出""数据采集"等）只是占位，务必替换为论文真实内容

- 避免照搬模板的阶段数量——依据论文真实研究环节增减阶段

- 工具/方法节点只列论文真实采用的方法，避免为好看而堆砌无关方法名

**6. 连线绝对不可穿过节点（全部 TikZ 图通用）**

- 若上一行有 3 个方法节点，下面有一行工具节点，再下面有 3 个方法节点，连线务必从上方方法节点的相应列垂直下来，避免从一个节点分叉斜穿工具行

- 无误做法：每列独立垂直连线（spec→mech, ident→hetero, est→ext），避免从 est 一个点分三叉

- 跨行连接走行间空白区域，用 `|-` 或 `-|` 直角拐弯

- 反馈箭头走图的最外侧边缘，避免穿过中间节点

**7. 变量关系图/框架图的防穿过模式（⛔ 最常出错的场景）**

变量关系图（自变量→因变量、控制变量→因变量）是线穿过文字的重灾区。务必遵循：

- **左侧控制变量连到右侧因变量时，避免斜线直连** — 斜线会穿过中间的标注文字和其他节点

- **无误做法 A（推荐）**：控制变量用一个汇聚点，从汇聚点画一条线到因变量

  ```latex

  % 控制变量汇聚后统一连线，不要每个控制变量单独斜线连过去

  \node[circle, fill=gray!30, minimum size=4pt, inner sep=0pt] (ctrl_hub) at (控制变量右侧, 因变量y) {};

  \foreach \x in {ctrl1, ctrl2, ctrl3} { \draw[arrow] (\x.east) -| (ctrl_hub); }

  \draw[arrow] (ctrl_hub) -- (因变量.west);

  ```

- **无误做法 B**：控制变量放在因变量正下方，垂直连线

- **无误做法 C**：控制变量用虚线框框起来，只画一条虚线箭头到因变量，标注"控制"

- **⛔ 错误做法**：3-5 个控制变量各自画一条斜线穿过全部图连到因变量 — 必然穿过中间的文字标注

- **固定效应/时间效应等标注文字**：放在箭头旁边用 `node[above]` 或 `node[below]`，避免放在箭头路线中间被穿过

- **假设标注（H1/H2/H3）**：放在箭头的上方或下方（`node[above, midway]`），避免放在两个节点之间的空白处被其他箭头穿过

**8. 几何示意图的标注防遮挡（⛔ 物理/工程类赛题常见）**

几何示意图（光学反射、传感器布局、力学分析等）的标注密集区域是遮挡重灾区。多个变量标注（坐标、角度、距离、法向量）集中在同一个点附近，务必遵循：

- **标注散射原则**：同一个点附近的多个标注务必分散到不同方向（上/下/左/右/左上/右下），不可都堆在一侧。用 `node[above right=3pt]`、`node[below left=5pt]` 等精确控制偏移

- **引线标注**：如果标注太多放不下，用引线（`\draw[thin, gray] (point) -- ++(0.8,0.5) node[right] {...}`）把标注拉到空白区域

- **标注优先级**：关键变量（如 `R`、`H`、`\alpha`）放在最显眼的位置，次要标注（如坐标 `(x_0, y_0)`）可放远一些

- **避免标注压在线/面上**：标注文字务必有白色背景（`fill=white, inner sep=1pt`），防止和底下的线条/填充色混在一起

- **角度标注用 arc**：角度 `\alpha`、`\theta` 用 `\draw[->] (point) ++(起始角:半径) arc (起始角:终止角:半径)` 画弧线标注，不要用文字直接放在角的位置

- **距离标注用双头箭头**：`H`、`R`、`d` 等距离用 `\draw[<->]` 双头箭头标注在图形外侧，避免标在图形内部和其他元素重叠

**9. 算法过程图的循环回路防遮挡**

算法过程图（遗传算法/模拟退火/梯度下降等）有循环回路箭头，容易和判定分支标签重叠：

- **循环回路走图的最外侧**：从判定节点的右侧出发（`.east`），向右走到图的最右边缘，再向上回到循环起点。避免从判定节点的左侧走——左侧一般情况下有"否"标签

- **循环回路和判定标签不可在同一侧**：若"是"标签在下方、"否"标签在右方，循环回路就从右方走；反之亦然

- **循环回路箭头用虚线**：`dashed` 样式区分于主过程的实线，视觉上不抢眼

- **循环次数/终止条件标注**：放在回路箭头的外侧（`node[right, midway]`），避免放在回路内侧和主过程节点重叠

**10. 网络图/拓扑图的边标签防遮挡**

网络图节点密集时，边上的权重/标签容易互相遮挡：

- **边标签用 `auto` 定位**：`node[auto, font=\scriptsize]` 让 TikZ 自动化选取标签放在边的哪一侧

- **交叉边用 `bend`**：两条边若交叉，在此之中一条用 `bend left=15` 或 `bend right=15` 绕开

- **密集区域只标注关键边**：若 >8 条边汇聚到一个节点，只标注权重最大/最小的几条，其余省略

- **节点间距不少于 2cm**：网络图节点不可太密，`node distance=2cm` 是最低要求

**11. 3D 透视图的标注定位**

3D 透视图（圆柱体/球体/曲面等）的标注需考虑透视变换：

- **标注放在图形轮廓外侧**：避免把标注放在 3D 物体的"表面上"——透视变换后位置会偏移

- **用引线从轮廓拉出**：`\draw[thin, ->] (surface_point) -- ++(1, 0.5) node[right] {$P_0$}`

- **坐标轴标注放在轴的末端**：`x`/`y`/`z` 轴标签放在箭头末端的外侧，避免放在轴线中间
