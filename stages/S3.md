# S3: Model Selection & Debate

**Goal**: Propose candidate models from 4 differentiated perspectives in parallel, eliminate fragile proposals through multi-round red-blue adversarial review, and have the review panel produce a quantitative final decision selecting the primary and backup models. Iterate if thresholds are not met.
**Dependency**: S2 (or S1 if S2 was skipped)
**Execution Style**: Temporary sub-agents write directly to their respective output files (isolated writes: the 4 proposal groups are mutually invisible); the main agent handles aggregation and iterative decision-making.

---

## Stage Principles

1. All sub-agents are temporary calls that write directly to their respective output files.
2. Non-compliance means rejection: if the review panel's score does not meet the threshold, iteration is mandatory; forced passing is not allowed.
3. Proposal groups must produce outputs in parallel, isolated from each other, and must not read each other's outputs.
4. At least 1 round of adversarial review is mandatory (Round 1 required). Round 2 is triggered when a REPAIR verdict exists. Round 3 only in Championship mode.
5. Iteration limit is 3; beyond that, generate a manual decision report; infinite loops are prohibited.
6. All proposal group sub-agents analyze based on existing literature in the `CUMCM_Workspace/_skill/references/` directory. If supplementary literature is needed, annotate `[Suggest supplement: keywords xxx]` in the output; the main agent performs web_search uniformly.

---

## Mode Branch Routing (P1-1 Streamlined)

After Step 1, the main agent selects the execution path based on the current mode:

| Mode | Proposal Groups | Attack/Defense Rounds | Final Review | Total Calls |
|------|----------------|----------------------|--------------|-------------|
| **Fast** | Proposer-A (1) | Round 1 simplified (Red->Referee, skip Blue) | Reviewer | **3** |
| **Standard** | Proposer-A + Proposer-B (2) | Round 1 full (Red->Blue->Referee) | Reviewer | **5** |
| **Championship** | Proposer-A/B/C/D (4) | Round 1 full + Round 2 conditional | Reviewer | **8** |

**Fast Mode Simplifications**:
- Only invoke Proposer-A (precision-oriented), skip B/C/D
- Round 1: Red Team attack -> Referee verdict (skip Blue Team defense)
- Referee verdict serves directly as final review, no separate Reviewer call
- Skip Step 6 (diversity check), Step 10/11 (Round 2/3)

**Standard Mode Simplifications**:
- Invoke Proposer-A + Proposer-B, skip C/D
- Round 1: full attack-defense (Red->Blue->Referee)
- Round 2 triggered only if REPAIR verdict exists
- Skip Step 11 (Round 3)

**Championship Mode**: Execute all Steps 2-14, no simplification.

### Step 13: Main Agent Synthesis and Iteration Decision

```

    -> Proceed to Step 14 (metric specification + innovation justification)

    -> Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S3 --reason "Reviewer panel requires fixes"
    -> Return to Step 7 (re-attack/defense for CONDITIONAL models)
    -> Max 2 iterations; beyond that, submit for manual decision

    -> Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S3 --reason "Reviewer panel rejected all proposals"
    -> Return to Step 2 (re-propose with failure reasons)
    -> Max 1 re-proposal; beyond that, submit for manual decision
```

- `methods/manual_decision_report.md`: Iteration history summary + failure reasons per round + recommended manual decision direction
