**Goal**: Complete the final paper build, ensuring full consistency and completeness.
**Dependency**: S9.5 (paper_final.md)
**Execution Style**: Writer (formatting) + Planner (consistency audit)
**Meeting**: None (main agent executes directly)

## Stage Principles (Not Negotiable)

1. **DO NOT skip any step.** All 9 steps must be executed.
2. **Consistency is core**: Paper-code-figures-tables must be completely consistent.
3. **Cross-subquestion consistency is a hard rule**: Variable names, units, and assumptions must be unified across all sub-questions.
4. **Evidence chain completeness**: Every conclusion must have traceable data support.
5. **Format compliance is the baseline**: All format checks must pass before delivery.

**Fast Mode Streamlined Path (P1-2)**: Skip Step 5 (cross-subquestion consistency check). The main agent only performs a quick evidence chain check in Step 6 (final audit). Remaining steps (code appendix, references, DOCX build, chapter-by-chapter check, format compliance) must still be executed.

## Step 1: Collect Code Appendix

## Step 2: Format References

Ensure references [1]-[N] are formatted per competition standards:
- Journal: `[No.] Author(s), Title, Journal Name, Volume(Issue): Year, Pages.`
- Book: `[No.] Author(s), Title, Publisher Location: Publisher, Year.`
- Web: `[No.] Author(s), Resource Title, URL, Access Date.`
- Check reference numbering continuity
- Check reference count meets target (Standard >= 10 / Championship >= 15)

---

## Step 3: Generate Table of Contents

Auto-generate the table of contents from paper titles. Ensure the TOC exactly matches the body titles and page numbers are correct.

## Step 4: Build DOCX

```bash
python CUMCM_Workspace/_skill/scripts/build_docx.py paper_final.md final.docx
```

**DOCX Build Check**:
- Fonts: HeiTi for titles, SongTi for body, Times New Roman for numbers
- Font sizes: Title size 3, Level 1 size 4, Body size small 4, Table size 5
- Line spacing: 1.5x or fixed 22pt
- Margins: 2.5cm all sides
- Three-line tables: top line 1.5pt, below header 0.5pt, bottom line 1.5pt, no vertical lines

## Step 8: Chapter-by-Chapter Completeness Check (MANDATORY)

**Check each chapter for the following**:
- [ ] Abstract: word count meets target, quantified result count, no vague words, numbers match body
- [ ] Problem Restatement: includes background + known + to-be-solved
- [ ] Problem Analysis: overall analysis + per-subquestion method selection rationale
- [ ] Model Assumptions: 3-5 items, each with justification
- [ ] Symbol Description: three-column table, no omissions
- [ ] Model Establishment & Solution: independent chapter per sub-question, including derivation + solution + results + D-A-C
- [ ] Unified Framework: from unified_kernel.md
- [ ] Sensitivity Analysis: parameter perturbation + baseline comparison + extreme scenarios
- [ ] Model Evaluation: 2+ advantages + 2+ limitations + extension directions
- [ ] Conclusion: answer each question + key values + limitations and outlook
- [ ] References: >= 10 (Standard) / >= 15 (Championship), correctly formatted
- [ ] Appendix: source code complete and runnable

## Gate Failure -> Auto Rework (Hard-Coded, Cannot Be Skipped)

```bash
IF gate_check returns FAIL:
    1. Record failed gate name and failure reason
    2. IF failed gate is cross_subquestion_consistency (cross-subquestion inconsistency):
        -> Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S5 --reason "S10 found cross-subquestion inconsistency: <reason>"
        -> Fallback to S5 to fix inconsistent sub-questions
        -> rework count +1 (counted in S10 rework total)
    3. IF failed gate is G6_audit_layer_passed (paper-code inconsistency):
        -> Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S8 --reason "S10 found paper-code inconsistency: <reason>"
        -> Fallback to S8 for paper revision
        -> rework count +1 (counted in S10 rework total)
    4. IF failed gate is G6_appendix_complete (appendix incomplete):
        -> Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S10 --reason "Appendix incomplete: <reason>"
        -> Return to Step 1 (re-collect code appendix)
        -> rework count +1
    5. If rework >= 2 times: log to state/decision_log.json, generate failure report, block and prompt human intervention
```

**This logic cannot be omitted. Final delivery failure must execute the rework command, record count, and block on exceed.**
