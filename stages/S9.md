# S9: Adversarial Review -- M004 Red-Team Review Meeting

**Goal**: Conduct multi-round adversarial review of the paper, eliminating P0/P1 issues.
**Dependency**: S8 (paper.md)
**Execution Style**: Critic (Red Team) -> Reviewer (Final Review) -> Writer (Revision)
**Meeting**: M004 (adversarial)
**Mode**: **Independent Sub-Agent Call (Platform-Agnostic)**

---

## Stage Principles (Not Negotiable)

1. **DO NOT skip any step.** All 6 steps must be executed.
2. **Championship mode must execute 3 rounds of review from different perspectives** (mathematical/domain/judge).
3. **Anti-AI trace is a P0 review item**: All AI cliches must be eliminated in this round.
4. **Revision verification must be item-by-item**: Every fix must be verified as actually repaired.
5. **Fundamental issues must fall back to S5**: Do not patch model defects at the paper level.
6. **Abstract-body consistency is a hard rule**: Any inconsistency triggers P0.

**Fast Mode Streamlined Path (P1-2)**: Execute only 1 review round -- Step 1 (Critic red-team review) -> Step 3 (revision plan) -> Step 4 (Writer revision) -> Step 5 (verification) -> Step 6 (gate). Skip Step 2 (Reviewer final review). Championship multi-round review is also skipped.

**P1-3 Quick-Fix Mode (auto-enabled when remaining time < 6 hours)**:
- Critic checks P0 items only (AI cliches + anonymity + numerical inconsistency), skip P1/P2
- Writer revision fixes P0 items only, no formatting/expression optimization
- Skip Step 5b (Championship multi-round review)
- Max 1 rollback (standard mode is 2)
- Target duration: 1-2 hours

```markdown
## Review Dimensions (Four-Dimension Weighted Scoring)

### 1. Mathematical Correctness (30%)
- Is formula derivation rigorous (each step justified)?
- Are symbols consistent (unified throughout)?
- Are assumptions reasonable and justified?
- Is model selection justified?

### 2. Reproducibility (25%)
- Is code reproducible (random seeds, dependency versions)?
- Are results stable (cross-validation consistency)?
- Is data processing transparent?

### 3. Logical Consistency (25%)
- Are abstract and body values exactly consistent?
- Are figures/tables and text descriptions consistent?
- Are conclusions and data consistent?
- Are variables/units/assumptions consistent across sub-questions?

### 4. Expression Standard (20%)
- Are three-line tables formatted correctly?
- Are figure/table titles conclusive (not descriptive)?
- Is reference format unified?
- Does the body have AI traces (34 type checklist)?

### 5. Template Format Compliance (Additional Check)
- Compare with templates in `templates/paper_templates/`, check each section's **structural format** compliance
- Focus on: abstract four elements present, assumptions with two-column justification, continuous formula numbering, three-part conclusion
- **Note**: Check format structure only; content consistency with template examples is not required

## Attack Strategies (All Must Be Executed)

### Attack 1: Assumption Attack
- For each assumption, ask: "Under what conditions does this assumption break?"
- Assess whether conclusions still hold when assumptions are relaxed

### Attack 2: Derivation Attack
- Check if each formula derivation has skipped steps
- Verify variable definitions are complete

### Attack 3: Data Attack
- Check if data sources are reliable
- Check if data processing is biased
- Check for data leakage

### Attack 4: Result Attack
- For each numerical result, ask: "Is this value reasonable?"
- Is the improvement over baseline significant?
- Is sensitivity analysis sufficient?

### Attack 5: Negative Result Honesty Check (MANDATORY)
- Check for evaded negative results
- Check for overclaiming
- Check if limitations are honestly stated

### Attack 6: AI Trace Check (Pre-review for S9.5)
- Scan paragraph by paragraph for 34 types of AI cliches
- Check sentence length variation (template-like AI patterns)
- Check paragraph structure variation
```


**Gate failure -> auto rework (hard-coded, cannot be skipped)**:
```bash
IF adversarial gate returns FAIL:
    1. Record failure reason (P0 not zeroed / P1 exceeded / AI traces not cleared / abstract inconsistent / revision verification failed)
    2. Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S9 --reason "Review gate failed: <reason>"
    3. Return to Step 3 (regenerate revision plan) -> Step 4 (Writer re-revision) -> Step 5 (re-verify)
    4. rework count +1
    5. If rework >= 2: log to state/decision_log.json, generate failure report, block and prompt human intervention
    6. If failure reason is "model/experiment fundamental issue":
        -> Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S5 --reason "S9 found fundamental issue: <reason>"
        -> Fallback to S5 for re-execution
```

**Hard criteria for "fundamental issue" (triggers S5 fallback)**:
- P0 issues involve model assumption errors or derivation flaws
- Core numerical results remain inconsistent after multiple revision rounds
- Broken evidence chain that cannot be fixed at the paper level

**This logic cannot be omitted. Review failure must execute the rework command, record count, and block on exceed. Championship mode S9 requires 3 rounds of review from different perspectives.**
