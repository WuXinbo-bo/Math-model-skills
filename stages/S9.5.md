## Step 1: Run Full Publication Check (MANDATORY)

```bash
python CUMCM_Workspace/_skill/scripts/publication_checker.py check outputs/paper/paper_revised.md --mode <mode>
```

**6 Dimensions of Checks**:

### Dimension 1: AI Trace Detection (P0 level, highest priority)

**34 Types of AI Cliches Check** (refer to nature-writing-guide.md Chapter 1 blacklist):

| Category | Typical Cliches | Quantity Limit |
|----------|---------------|----------------|
| Cliches | This paper addresses/firstly secondly finally/has important significance/in summary | 0 occurrences |
| Over-modification | significantly improved/obviously improved/greatly enhanced/fully considered | 0 occurrences |
| Vague quantifiers | good performance/excellent results/high precision/small error | 0 occurrences |
| Generic connectors | it is worth noting/it is not difficult to find/as is well known/on this basis | 0 occurrences |
| Causal assertions | experiments prove/results show/fully demonstrate/strongly support | <= 2 occurrences |

**Structural AI Trace Check**:
- [ ] Is paragraph length varied (no 3+ consecutive paragraphs of the same length)?
- [ ] Is sentence length varied (no 3+ consecutive sentences of the same length)?
- [ ] Is paragraph structure varied (argumentative/narrative/comparative mixed)?
- [ ] Are there parallel structure triples?
- [ ] Is each paragraph's expression style unique?

### Dimension 2: Informal Expression Detection (P1 level)

| Type | Examples | Treatment |
|------|----------|-----------|
| First person | We think/We find | Change to "This paper"/"Experimental data" |
| Colloquial | roughly/about/mostly okay | Change to precise values |
| Subjective judgment | Personally I think | Delete or replace with data-supported statements |
| Non-academic abbreviations | ASAP/got it/works | Change to formal expressions |

### Dimension 3: Figure-Text Layout Standards (P1 level)

- [ ] Each figure/table appears after its first citation (text-before-figure)
- [ ] Figure/table citation completeness (each figure has D-A-C interpretation in body)
- [ ] Figure/table title position correct (figure title below, table title above)
- [ ] Figure/table numbering continuous (Figure 1 -> Figure 2 -> ... -> Figure N, no gaps)

### Dimension 4: Format Compliance (P1 level)

Compare with high-scoring paper PDFs in `templates/paper_templates/` (select 1-2, read with read_file), check each section's structure:

- [ ] Abstract has all four elements (background + method + result + features)
- [ ] Assumption table has two-column justification
- [ ] Formula numbering continuous
- [ ] Symbol table complete (each variable has meaning and unit)
- [ ] Three-line table format correct (top line 1.5pt, below header 0.5pt, bottom line 1.5pt, no vertical lines)
- [ ] Section hierarchy correct (level 1 -> level 2 -> level 3)
- [ ] Units unified (same physical quantity uses same unit throughout)
- [ ] Font unified (Song/Times New Roman/Hei)
- [ ] Figure/table titles are conclusive (not descriptive)
- [ ] Conclusion is three-part (summary + limitations + outlook)
- [ ] Reference format unified (GB/T 7714-2015)

### Dimension 5: Academic Standards (P1 level)

- [ ] No dangling references (cited references all appear in reference list)
- [ ] Citation format unified (all use GB/T 7714-2015)
- [ ] Terminology consistent (same concept uses same term throughout)
- [ ] Symbols consistent (same variable uses same symbol throughout)

### Dimension 6: Competition-Specific Standards (P0 level)

- [ ] Anonymity: no school name, advisor name, team member names
- [ ] Word count meets target: CUMCM >= 12K / MCM >= 8K words
- [ ] Figure count meets target: >= 6 (Standard) / >= 10 (Championship)
- [ ] Reference count meets target: >= 10 (Standard) / >= 15 (Championship)
- [ ] No placeholders: no TBD/TODO/pending
- [ ] Abstract does not exceed one page (CUMCM abstract <= 1 page)

## Step 2: Issue Grading and Handling

**P0 Issues (all must be zeroed = 0)**:
- AI cliches cumulative >= 3 occurrences
- Generic connectors cumulative >= 5 occurrences
- Vague quantifiers cumulative >= 5 occurrences
- First person in abstract
- Anonymity violation (leaking school/name)
- Word count below target
- Placeholder remnants
- Abstract values inconsistent with body

**P1 Issues (must fix to <= 2 remaining)**:
- Over-modification
- Colloquial expressions
- Figure-text layout errors
- Numbering discontinuities
- Terminology inconsistency
- Dangling references

**P2 Issues (recommended to fix, <= 5 remaining)**:
- Non-academic abbreviations
- Minor formatting issues
- Readability optimization

## Step 5: Championship Mode Extra Review (Championship Only)

**Round 1 -- Language Perspective (Paragraph-by-Paragraph Scan)**:
- Read paragraph by paragraph, flag any "AI-like" expressions
- Check sentence length variation: 3+ consecutive sentences of same length -> break apart
- Check paragraph structure variation: consecutive use of same pattern -> replace
- Ensure each paragraph has a unique expression style
- Check that logical connections between paragraphs are natural

**Round 2 -- Judge Perspective (Quick Scan)**:
- Pretend to be a CUMCM judge, quick-scan the full paper in 5 minutes
- Flag any "unprofessional" or "perfunctory" paragraphs
- Check if figures are "worth a thousand words"
- Check if the abstract can grab the judge's attention within 30 seconds

**Round 3 -- Anti-AI Trace Specialized Scan**:
- Scan item by item using the 34-type blacklist
- Check for missed AI cliches
- Check if replaced expressions are natural
