# S7: Evidence Chain & Traceability

**Stage Goal**: Link all key conclusions, figures, tables, code outputs, and assumptions into a traceable evidence chain, ensuring every number, judgment, and conclusion in the paper can be traced back to specific evidence.

**Dependency**: `S6`

**Execution Principles**:
- All sub-agents are temporary calls that **write directly to their respective output files**.
- The evidence chain is not an appendix decoration; it is the core of paper credibility.
- Any conclusion that cannot be traced must be explicitly tagged; do not disguise it as verified.

## Step 1: Invoke Planner Sub-Agent

**Complete instruction sent to sub-agent**:

```text
You are the Planner temporary evidence chain agent, existing only in this round. Complete evidence chain organization and consistency check for the S7 stage.

## Role Constraints
- You are responsible for evidence organization and reasoning; write results directly to the specified output file.
- You must clearly indicate the evidence source for each conclusion.

## File Permissions
- Readable: outputs/q{N}/*, outputs/verification/*, outputs/paper/*
- Writable: outputs/evidence/Q{N}/conclusion_evidence_map.md, outputs/evidence/Q{N}/paper_code_consistency_report.md, outputs/evidence/Q{N}/assumption_evidence_matrix.md
- Forbidden: all other files
- Content without evidence support must be explicitly tagged as "unconfirmed" or "not verifiable."

## Required Inputs
1. `outputs/q{N}/summary.md`
2. `outputs/q{N}/tables/`
3. `outputs/q{N}/figures/`
4. `outputs/verification/cross_validation.md`
5. `outputs/verification/constraint_checks.md`

## Task Objectives
1. Establish conclusion-to-evidence mapping.
2. Establish assumption-to-verification mapping.
3. Check consistency between paper numbers and code outputs.
4. Tag unverified claims and weak evidence.
5. Provide evidence chain completeness conclusion.

## Output Requirements
Output in Markdown:
- Evidence directory
- Conclusion mapping table
- Assumption verification table
- Consistency issues
- Items needing supplementary evidence
- Final traceability judgment
```

## Required Actions

Check items (main agent verifies item by item, no external script dependency):
- [ ] `conclusion_evidence_map.md` exists and is non-empty
- [ ] `paper_code_consistency_report.md` exists and is non-empty
- [ ] All assumptions have verification evidence
- [ ] Unmapped conclusion proportion within acceptable range (unmapped < 10%)
- [ ] Code-paper inconsistencies = 0


## Gate Failure -> Auto Rework (Hard-Coded, Cannot Be Skipped)

```bash
IF evidence-chain gate returns FAIL:
    1. Record failure reason (which conclusions lack evidence, which assumptions unverified, which data inconsistent)
    2. Run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S7 --reason "Evidence chain incomplete: <reason>"
    3. Return to Step 5 (identify unverified claims) -> Step 6 (re-execute evidence chain check)
    4. rework count +1
    5. If rework >= 2: log to state/decision_log.json, generate failure report
    6. If the failure reason is "upstream data missing": run python CUMCM_Workspace/_skill/scripts/stage_executor.py rework S5 --reason "S7 found broken evidence chain: <reason>", fallback to S5
```

**Hard criteria for "broken evidence chain" (auto-triggered)**:
- Any key numerical conclusion cannot be traced to a specific code output
- Any assumption has no verification evidence
- Code-paper inconsistencies > 0
- Unmapped conclusion proportion > 10%

**This logic cannot be omitted. Evidence chain failure must execute the rework command, record count, and block on exceed.**
